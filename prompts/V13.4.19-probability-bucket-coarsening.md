# 10. Coarsening Strategy

必须实现至少 3 套粗化方案。

---

## 10.1 Scheme A - Remove Time Buckets

如果当前 probability table 包含：

```text
timeOfDay
dayOfWeek
```

Scheme A 移除这些字段。

目的：

```text
增加每个 bucket 的样本数，避免时间维度导致桶过碎。
```

---

## 10.2 Scheme B - Merge RSI / EMA / Bollinger Buckets

粗化：

```text
RSI:
below30
30-45
45-55
55-65
above65
```

合并为：

```text
low
middle
high
```

EMA distance：

```text
below_ema20
near_ema20
extended_above_ema20
```

可保持或合并为：

```text
near_or_below
extended
```

Bollinger position：

```text
lower
middle
upper
outside
```

合并为：

```text
lower_or_middle
upper_or_outside
```

---

## 10.3 Scheme C - Regime + Liquidity + BTC Only

只保留核心维度：

```text
regime
liquidityBucket
btcState
```

目的：

```text
测试最粗粒度的统计结构是否仍然没有正期望。
```

---

## 10.4 Scheme D - Regime + Module + Volatility

如果可用，保留：

```text
regime
moduleType
volatilityBucket
btcState
```

用于判断：

```text
trend module 与 mean_reversion module 是否存在统计差异。
```

# 11. Gate Definitions


---

## 11.1 Current Gate

保持 V13.4.14 原 gate，不修改：

```text
sampleCount >= 50
hitTpBeforeSlProbability >= 0.45
profitFactor >= 1.2
expectancy > 0
```

---

## 11.2 Research Gate

用于研究，不用于交易：

```text
sampleCount >= 100
profitFactor > 1.05
expectancy >= 0
```

或：

```text
sampleCount >= 50
profitFactor >= 1.1
expectancy > 0
```

必须写明：

```text
researchGate is not used for trading.
```

---

## 11.3 Exploratory Gate

仅用于发现可能值得调查的桶：

```text
sampleCount >= 30
profitFactor > 1.0
```

必须写明：

```text
exploratoryGate is for analysis only.
Do not connect to strategy entry.
```
