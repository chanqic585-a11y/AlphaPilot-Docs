# 3. Core Goal

V13.4.4 的核心目标：

```text
1. 保留 V0.1 baseline 不变
2. 创建 V0.2A/B/C/D/E 独立候选策略副本
3. 在同一 pairs / timerange / config 下执行对比回测
4. 输出每个候选的真实 Freqtrade backtest report
5. 转换为 AlphaPilot 标准报告
6. 生成 comparative report
7. 判断哪些候选值得进入下一轮更大范围回测
8. 不批准 Dry-run
9. 不接实盘
```

一句话目标：

```text
用真实回测数据比较 V0.1 和 V0.2 候选，而不是凭感觉决定怎么改策略。
```

# 10. Candidate A - Trend Strict

类名：

```text
AlphaPilotVolumeReboundV02ATrendStrict
```

目标：

```text
增强 4h 趋势过滤，减少弱趋势下的反弹失败。
```

修改方向：

```text
V0.1 4h filter:
  close_4h >= ema200_4h * 0.98

V0.2A:
  close_4h >= ema200_4h
```

可选附加条件：

```text
如果实现简单且不破坏稳定性，可增加：
ema20_4h >= ema200_4h * 0.995
```

但请避免一次加入过多变化。

建议 V13.4.4 首选只测试：

```text
close_4h >= ema200_4h
```

原因：

```text
单变量变化更容易解释。
```

# 12. Candidate C - Exit Cleanup

类名：

```text
AlphaPilotVolumeReboundV02CExitCleanup
```

目标：

```text
重新评估 MACD 两根走弱退出，因为 V13.4.1 中该 exit reason 显著亏损。
```

推荐 V13.4.4 采用一个清晰候选：

```text
MACD weak exit only when current_profit > 0
```

即：

```text
只有当前 trade 已浮盈时，才允许 macd_histogram_two_candle_weakness 触发退出。
```

这样可以测试：

```text
MACD weak exit 是否适合做盈利保护，而不是亏损退出。
```

如果 Freqtrade callback 获取 current_profit 不稳定，可以退而求其次：

```text
禁用 MACD weak exit
```

但必须在报告中说明实际采用哪种。

# 20. Comparison Gate

不要只按 total return 判断。

候选通过初筛至少需要：

```text
1. profitFactor > baseline profitFactor
2. maxDrawdownPct < baseline maxDrawdownPct
3. maxConsecutiveLosses <= baseline maxConsecutiveLosses
4. totalReturnPct > baseline totalReturnPct
5. tradeCount 不得过低到无意义
```

但即使通过，也不自动 Dry-run。

报告必须写：

```text
dryRunApproved = false
```

除非用户后续明确授权进入 Dry-run，并且至少完成更长区间回测。
