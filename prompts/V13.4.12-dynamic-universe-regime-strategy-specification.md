# AlphaPilot 新策略主线：Dynamic Universe + Regime Router + Probability Score

> 文档用途：重新定义 AlphaPilot 后续策略研究主线。  
> 当前结论：旧的 Volume Rebound V0.1/V0.2 与 Trend Pullback 1H V0.1 已经暴露出泛化不足、币种池过宽、交易成本敏感、实盘执行可行性不足等问题。后续不应继续小幅调参，而应切换到新的策略框架。  
> 新主线名称：**AlphaPilot Dynamic Regime Strategy V0.1**  
> 中文名称：**AlphaPilot 动态市场状态策略 V0.1**

---

## 1. 为什么要换新策略主线

之前已经验证过两条主要方向：

```text
1. 15m 放量反弹策略 Volume Rebound V0.1 / V0.2
2. 1h 趋势回调策略 Trend Pullback 1H V0.1
```

真实回测结果显示：

```text
Volume Rebound V0.1/V0.2：
Top30 扩大验证后深度负收益，滑点后更差。

Trend Pullback 1H V0.1：
BTC/ETH/SOL 小样本 smoke 为正，
但 Top30 6个月扩大验证后失败。
```

核心问题不是单一参数，而是：

```text
1. 固定 Top30 太宽，很多币不适合同一个策略
2. 加密货币轮动快，热点变化快
3. 山寨币噪音大、流动性差异大
4. 趋势策略和均值回归策略不能混用同一套条件
5. 交易成本和滑点影响极大
6. 回测盈利不代表真实执行可行
7. 当前策略缺少动态币种池、市场状态识别和统计概率验证
```

因此，后续不应继续：

```text
固定 Top30
+
固定单一指标组合
+
直接全市场套用
```

而应该切换为：

```text
动态币种池
+
市场状态路由
+
趋势 / 均值回归模块
+
统计概率评分
+
流动性闸门
+
风险闸门
```

```text
AlphaPilot Dynamic Regime Strategy V0.1
```

中文：

```text

```

核心定义：

```text

再用统计概率评分、流动性闸门和 Risk Gate 决定是否进入回测交易。
```

## 4. 核心模块一：动态币种池 Dynamic Universe

### 4.1 目标

动态币种池解决的问题：

```text
不是所有 Top30 币种都适合交易。
每天 / 每3天真正活跃、有流动性、有行情结构的币才进入候选池。
```

### 4.2 第一版 Universe 名称

```text
DynamicUniverseV01
```

### 4.3 入池基础条件

```text
1. OKX USDT 永续
2. 非稳定币
3. 非即将下架币
4. 历史数据至少 90 天
5. K线缺失率低
6. 24h quoteVolume 达标
7. 3d average quoteVolume 达标
8. bid/ask spread 不过大
9. 不在异常风控黑名单
```

### 4.4 排名因子

动态池不只看涨幅榜，而是综合：

```text
1. 24h quoteVolume rank
2. 3d quoteVolume rank
3. 24h absolute return rank
4. 3d absolute return rank
5. 3d volatility rank
6. volume expansion rank
7. liquidity stability rank
```

### 4.5 第一版选择数量

建议：

```text
Top10
```

最多：

```text
Top15
```

不建议第一版直接 Top30 或全市场。

原因：

```text
Top30 在旧策略中已经证明噪音太多；
Top10 / Top15 更适合快速验证动态策略是否有效。
```


## 6. 核心模块二：市场状态识别 Market Regime Router

### 6.1 Regime 类型

```text
trend
mean_reversion
breakout
avoid
```

第一版实现：

```text
trend
mean_reversion
avoid
```

breakout 放到后续版本。

### 6.2 Regime Router 作用

```text
同一个币，不同市场状态下应使用不同策略。
趋势环境用趋势模块；
震荡/偏离环境用均值回归模块；
风险环境直接 avoid。
```

---

## 7. Regime 判断草案

### 7.1 Trend Regime

判断条件示例：

```text
4h close > EMA200
4h EMA20 > EMA50
1h close > EMA50
BTC 1h / 4h 不弱
ADX 或趋势强度达到阈值
```

适合：

```text
趋势回调延续
突破回踩确认
```

### 7.2 Mean Reversion Regime

判断条件示例：

```text
4h 不是强空头
1h 价格偏离 EMA20 / Bollinger middle 较远
RSI 短期过低
波动率适中
BTC 没有急跌
没有重大风险事件
```

适合：

```text
短期过度偏离后的均值回归
```

### 7.3 Avoid Regime

触发条件：

```text
BTC 急跌
4h 强空头
点差过大
流动性不足
成交量异常
重大新闻 / 宏观事件前后
价格过度拉升
数据缺失
```

Avoid 结果：

```text
不生成交易信号。
```

## 8. 核心模块三：趋势模块 Trend Module

### 8.1 模块名称

```text
TrendContinuationModuleV01
```

### 8.2 策略思想

```text
只在趋势明确的币种中，等待价格回调到合理位置，再等待重新走强确认。
```

### 8.3 初始逻辑草案

```text
1. Regime = trend
2. 4h 趋势向上
3. 1h 回调到 EMA20 / EMA50 区域
4. 1h 重新站上 EMA20
5. MACD histogram 改善
6. volumeRatio 健康
7. no-chase 通过
8. BTC safe
9. Liquidity Gate 通过
10. Probability Score 通过
```

### 8.4 出场方向

不再使用简单 1:1。

初始建议：

```text
stoploss：-2% 到 -2.5%
take profit：+4% 到 +5%
minimum reward/risk：>= 1.5R
time stop：8-12 根 1h
盈利状态下 momentum exit
```

## 9. 核心模块四：均值回归模块 Mean Reversion Module

### 9.1 模块名称

```text
MeanReversionModuleV01
```

### 9.2 策略思想

```text
只在非崩盘、非强空头环境下，寻找短期过度偏离后的回收机会。
```

### 9.3 初始逻辑草案

```text
1. Regime = mean_reversion
2. 4h 不是强空头
3. BTC 没有急跌
4. 价格跌破或接近 Bollinger lower
5. RSI 低位但开始回升
6. close 回收到 Bollinger lower 上方
7. volume 不萎缩
8. ATR 不过高
9. Liquidity Gate 通过
10. Probability Score 通过
```

### 9.4 出场方向

建议：

```text
take profit：回到 Bollinger middle 或 +2.5% / +3%
stoploss：结构破位或 -2%
time stop：短一些，避免接飞刀
```

均值回归不追求大趋势，重点是：

```text
快速修复
低持仓时间
严格止损
```

## 11. 核心模块六：统计概率评分 Probability Score

### 11.1 为什么需要概率评分

之前失败策略的问题是：

```text
指标条件通过，但不代表有统计优势。
```

所以每个候选信号都要问：

```text
历史上类似条件下，未来先到止盈的概率是多少？
```

### 11.2 标签定义

对每个候选信号，统计未来窗口：

```text
未来 8 根 K线
未来 12 根 K线
未来 24 根 K线
```

记录：

```text
是否先达到 take profit
是否先达到 stoploss
最大浮盈 MFE
最大浮亏 MAE
最终收益
持仓时间
```

### 11.3 条件桶

按以下条件分桶：

```text
regime
pair liquidity bucket
volume rank bucket
volatility bucket
RSI bucket
distance to EMA20 bucket
distance to Bollinger bucket
BTC state
time of day
day of week
```

### 11.4 概率评分规则

输出：

```text
sampleCount
hitTpBeforeSlProbability
averageReturn
averageLoss
profitFactor
expectancy
confidenceLevel
```

初始通过条件：

```text
sampleCount >= 50
hitTpBeforeSlProbability >= 0.45
profitFactor >= 1.2
expectancy > 0
```

如果样本不足：

```text
observe_only
```

## 14. 新策略执行流程

```text
1. 生成历史动态币种池快照
2. 对每个 pair 计算 market regime
3. regime = trend → 走 Trend Module
4. regime = mean_reversion → 走 Mean Reversion Module
5. regime = avoid → 不交易
6. 生成候选信号
7. 计算 Probability Score
8. 通过 Liquidity Gate
9. 通过 Risk Gate
10. 进入回测交易
```

## 24. 为什么这个新主线更合理

加密货币市场特点：

```text
币种多
轮动快
热点变化快
流动性分化明显
山寨币噪音大
不同市场状态适合不同策略
交易成本影响显著
```

因此更合理的系统不是：

```text
一个固定指标组合跑所有币
```

而是：

```text
先选活跃且可交易的币
再识别市场状态
再选择合适模块
再用统计概率确认
再过流动性和风控
```
