# AlphaPilot V12.2 - AI Technical Decision Copilot

## Goal

基于当前 `trade-discipline-journal` V12.1 版本，开发 AlphaPilot V12.2：AI Technical Decision Copilot。

V12.2 的目标是让 AI 深度参与用户交易决策前的分析工作。

V12.2 不是自动交易系统，不是实盘交易系统，不是下单系统。

V12.2 的目标是：

```text
用户输入加密货币名称
系统自动读取本地行情和指标数据
系统自动检索 V12.1 技术分析知识库
系统自动构建 AI 分析上下文
AI 自动生成中文技术分析报告
AI 自动给出技术偏向、情景推演、风险条件、重点观察
AI 自动分析用户交易习惯
系统自动保存 AI 报告、证据链、习惯观察和用户反馈
用户只负责最终判断和实际操作
```

V12.2 要尽量减少人工参与。

用户只需要：

```text
输入币种
点击开始分析
阅读 AI 报告
判断是否有参考价值
实际操作由用户自己完成
给 AI 报告反馈
```

AI 负责：

```text
快速分析
整理证据
引用 V12.1 技术知识
解释指标
发现风险
记录习惯
生成报告
保存数据
```

人工负责：

```text
最终判断
是否采纳
是否操作
如何操作
承担实际交易决策
```


```text

Evidence before conclusion.
Chinese-first mobile UX.

Scenario view, not order instruction.
Habit observation, not blame.
Reports before automation.
Feedback before V13.
```

## AI Boundary

V12.2 的 AI 可以深度参与分析，但不能拥有交易决策权。

### AI 可以输出

```text
当前技术偏向：偏多 / 偏空 / 震荡 / 方向不明确
趋势结构：增强 / 减弱 / 震荡 / 未确认
动能状态：增强 / 衰减 / 背离 / 冲突
支撑压力：接近支撑 / 接近压力 / 区间内运行
指标共振：RSI、MACD、成交量、趋势是否一致
指标冲突：哪些指标互相矛盾
偏多情景
震荡情景
偏空情景
风险条件
重点观察
数据质量说明
交易习惯观察
```

### AI 不能输出

```text
现在买入
现在卖出
立即做多
立即做空
开仓
平仓
下单
执行交易
仓位建议
杠杆建议
具体止损价
具体目标价
```

### 推荐替代表达

不要写：

```text
建议买入 BTC
```

应该写：

```text
当前技术结构偏强，但仍需要观察上方压力区是否被有效突破。
```

不要写：

```text
现在可以做空
```

应该写：

```text
如果价格跌破关键支撑区，并且成交量放大、RSI 继续走弱，则偏空情景的研究证据会增强。
```

不要写：

```text
止损放在 42000
```

应该写：

```text
需要关注关键支撑区失守后的结构变化，具体交易计划由用户自行判断。
```

## Chinese-first UI Requirement

V12.2 的 UI 必须偏向中文使用体验。

目标用户是中文使用者，手机端实测时需要快速阅读 AI 分析结果，所以页面标题、按钮、提示文案、分析报告、反馈选项应以中文为主。

### UI Language Rules

1. 页面主语言使用简体中文。
2. 技术指标缩写可以保留英文：

   * RSI
   * MACD
   * ATR
   * VWAP
   * ADX
   * K-line
3. 英文指标后面尽量配中文解释：

   * RSI 动能状态
   * MACD 趋势动能
   * ATR 波动率
   * VWAP 成交量加权均价
   * ADX 趋势强度
4. 不要大段英文报告。
5. AI 分析报告默认中文输出。
6. 知识引用可以保留英文标题，但必须给中文摘要。
7. 操作按钮使用中文。
8. 手机端阅读优先，减少复杂英文术语堆叠。

页面入口：

```text
我的 -> AI 分析
```

页面标题：

```text
AI 技术分析助手
```

副标题：

```text
输入币种后，AI 会结合行情、技术指标、知识库和交易习惯，生成结构化分析报告。
```

## AI Provider Layer

新增：

```text
src/ai-analysis/aiProvider.ts
```

必须支持两种 provider：

### 1. Mock Provider

默认启用。

用于无 API Key 时本地测试。

Mock Provider 必须返回结构化中文报告，方便页面和数据库测试。

### 2. OpenAI-compatible Provider

支持通过配置调用 OpenAI-compatible chat completion API。

要求：

```text
不要硬编码 API Key
不要 commit API Key
不要把 AI API Key 写入源码
不要把交易所 API Key 混入 AI 配置
不要接交易所私有权限
```

V12.2 的 AI Key 处理建议：

```text
开发测试阶段可以让用户在页面临时输入 AI API Key
默认只保存在 React state / memory
不要保存到 SQLite
不要写入 git
不要写入 README 示例真实值
```

如果项目已有安全存储能力，可以预留 SecureStore 方案；如果没有，不要新增复杂依赖，先使用 memory-only。

### Provider Settings

新增 `AIProviderSettings` 表可以保存：

```text
providerName
baseUrl
modelName
enabled
lastTestStatus
lastTestAt
```

但不要保存 raw API key。

允许保存：

```text
apiKeyStored: 0/1
```

不允许保存：

```text
apiKey
apiSecret
rawKey
secretKey
privateKey
```

## AI Output Structure

AI 输出必须尽量是 JSON。

新增类型：

```ts
export interface AIAnalysisStructuredOutput {
  language: 'zh-CN';
  technicalBias: '偏多' | '偏空' | '震荡' | '方向不明确';
  biasStrength: number;
  confidenceScore: number;
  dataQualityScore: number;
  summary: string;
  marketSnapshot: string;
  klineContext: string;
  trendContext: string;
  rsiAnalysis: string;
  macdAnalysis: string;
  atrVolatilityAnalysis: string;
  supportResistanceAnalysis: string;
  volumeAnalysis: string;
  chanTheoryContext: string;
  indicatorConfluence: string;
  indicatorConflict: string;
  bullishScenario: string;
  neutralScenario: string;
  bearishScenario: string;
  riskConditions: string[];
  whatToWatchNext: string[];
  habitObservations: string[];
  evidenceChunkIds: string[];
  limitations: string[];
  finalNote: string;
}
```

字段说明：

```text
technicalBias：只允许 偏多 / 偏空 / 震荡 / 方向不明确
biasStrength：0-100，表示技术偏向强度，不是胜率
confidenceScore：0-100，表示 AI 对分析完整度的信心，不是盈利概率
dataQualityScore：0-100，来自本地数据质量，不是信号强度
summary：中文总结
riskConditions：风险条件，不是止损建议
whatToWatchNext：观察点，不是交易指令
habitObservations：交易习惯观察，不是批评用户
evidenceChunkIds：引用的 V12.1 KnowledgeChunks
finalNote：必须包含“本报告是技术研究结果，不是自动交易指令”
```

```text

9. create AIAnalysisReport
```

```text

fallback report status = fallback
不能因为 AI 请求失败导致页面崩溃
```
