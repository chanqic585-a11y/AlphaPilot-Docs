## 12.1 4h Trend Filter

候选规则：

```text
4h close > 4h EMA200
4h EMA20 >= 4h EMA50
4h EMA20 slope >= 0
```

目的：

```text
只在大周期不弱的环境下寻找回调延续。
```

---

## 12.2 BTC Market Safety Filter

候选规则：

```text
BTC 1h 最近 3 根累计跌幅 > -1.5%
BTC 4h close 不低于 EMA200
BTC 1h MACD histogram 不连续恶化
```

目的：

```text
避免 BTC 急跌时做山寨币反弹。
```

---

## 12.3 1h Pullback Location

候选规则：

```text
当前价格接近 1h EMA20 / EMA50 区域
或 close 位于 Bollinger middle 附近
或距离近期支撑区域不远
```

建议第一版可实现为：

```text
close >= EMA50
close <= EMA20 * 1.015
close >= EMA20 * 0.985
```

目的：

```text
不是追高，而是在趋势中的合理回调区域等待。
```

---

## 12.4 Reclaim / Confirmation

候选规则：

```text
1h close > EMA20
或 1h close 从 EMA20 下方重新站上 EMA20
或 MACD histogram 当前值 > 前一根
```

第一版建议：

```text
close > EMA20
macd_hist > macd_hist.shift(1)
```

目的：

```text
避免只因为价格低就入场，必须出现重新走强。
```

---

## 12.5 Volume Quality

候选规则：

```text
volumeRatio >= 1.2 或 1.3
```

注意：

```text
1h 周期下 volumeRatio 不宜直接沿用 15m 的 1.5。
V13.4.7 只定义候选，V13.4.8 可以测试 1.2 / 1.3 / 1.5。
```

第一版推荐：

```text
volumeRatio >= 1.2
```

原因：

```text
1h 过滤更多，成交量阈值不宜过高导致样本过少。
```

# 13. V03 Exit Logic Draft

V0.1/V0.2 的 1:1 payoff 失败。

V03 应该避免继续使用纯：

```text
-3% stoploss
+3% take profit
```

推荐 V03 第一版候选：

```text
Stoploss: -2.5% 或 ATR-based stop
Take profit: +5%
Minimum reward/risk: 1.5R 或 2R
Time stop: 8 根 1h K线未盈利退出
Momentum exit: 只在盈利时启用
```

V13.4.7 必须明确：

```text
V03 不再默认 1:1 payoff。
```

推荐先写两个 exit profile 候选：

```text
ExitProfileA:
  stoploss -2.5%
  take profit +5%
  time stop 8h

ExitProfileB:
  ATR stop = entry - 1.5 * ATR
  take profit = 2R
  time stop 8h
```

V13.4.8 可优先实现 ExitProfileA，因更简单。

# 14. V03 Position Sizing

继续使用风险倒推仓位：

```text
riskAmount = accountEquity × riskPerTradePct
effectiveStopDistance = stopLossPct + feeRate × 2 + slippageRate × 2
positionNotional = riskAmount / effectiveStopDistance
requiredMargin = positionNotional / leverage
```

默认：

```text
riskPerTrade = 1%
leverage = 5x configurable
marginMode = isolated
```

但 V13.4.7 不写实盘执行代码。
