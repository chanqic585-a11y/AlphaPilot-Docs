# 10. Slippage-Adjusted Metrics

V13.4.4 已知：

```text
slippageApplied=false
```

V13.4.5 必须新增 post-processing 滑点调整指标。

默认滑点：

```text
slippageRate = 0.0005
即 0.05% one-way
```

对每笔交易计算：

```text
slippageCostPct = slippageRate * 2
```

如果使用 profit ratio：

```text
slippageAdjustedProfitRatio = rawProfitRatio - slippageRate * 2
```

如果使用 USDT profit：

```text
slippageCost = abs(positionNotional) * slippageRate * 2
slippageAdjustedProfit = rawProfit - slippageCost
```

如果无法准确获得 positionNotional，则使用可用字段估算，并写入 warning：

```text
slippageCostEstimated = true
```

报告必须明确：

```text
slippageAppliedByFreqtrade = false
slippageAppliedByPostProcessing = true
```

不要把 post-processing 滑点调整伪装成 Freqtrade 原生滑点。

# 13. Expanded Gate

候选不能只看收益。

V13.4.5 的 expanded gate 建议：

```text
1. slippageAdjustedTotalReturnPct > baseline slippageAdjustedTotalReturnPct
2. slippageAdjustedProfitFactor > baseline slippageAdjustedProfitFactor
3. maxDrawdownPct < baseline maxDrawdownPct
4. maxConsecutiveLosses <= baseline maxConsecutiveLosses
5. tradeCount >= 50
6. no single pair dominates loss/profit excessively
```

即使某候选通过 expanded gate：

```text
dryRunApproved 仍然必须为 false
```

原因：

```text
需要 V13.4.6 或 V13.5 前再做更长周期/稳定性验证。
```
