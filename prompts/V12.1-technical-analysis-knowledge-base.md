# AlphaPilot V12.1 - Technical Analysis Knowledge Base

## Goal

基于当前 `trade-discipline-journal` V12 版本，开发 AlphaPilot V12.1：Technical Analysis Knowledge Base。

V12.1 的目标不是接真实 AI，不是生成买卖建议，不是自动交易，不是实盘交易。

V12.1 的目标是建立 AlphaPilot 的本地技术分析知识库，为后续 V12.2 Real AI Analysis Copilot 做准备。

V12.1 要重点覆盖：

```text
K线分析
RSI 分析
MACD 分析
ATR / 波动率分析
布林带分析
成交量分析
压力位 / 支撑位分析
趋势结构分析
Pivot Points
VWAP
ADX 趋势强度
多周期分析
指标共振
缠论结构笔记
形态确认
技术分析失效场景
```

V12.1 应该减少重复的交易权限提醒、API Key 提醒、自动交易提醒。

内容比例建议：

```text
70% 技术分析知识
20% 技术分析验证 / 回测 / 指标局限
10% 风控和执行边界
```

V12.1 只做：

```text
本地知识库
技术分析资料源 metadata
AlphaPilot 原创技术分析知识切片
用户手动添加研究笔记
用户手动添加公开资料 metadata
用户手动粘贴开源摘要
本地关键词搜索
标签过滤
证据链关联
Research Dashboard 集成
```

V12.1 不做：

```text
真实 AI 分析
OpenAI / Claude / Gemini 接入
自动爬取网页全文
自动下载文件
自动生成买卖建议
自动创建 SignalCandidate
自动创建 ExecutionIntent
自动创建 PaperOrder
自动执行 Dry Run
真实交易
自动交易
```

## Product Definition

V12.1 是：

```text
Technical Analysis Knowledge Base
```

它不是：

```text
AI Trading Analyst
Trading Signal System
Buy/Sell Recommendation Engine
Auto Trading System
Live Trading System
```

V12.1 的主要价值：

1. 把 K线、RSI、MACD、ATR、支撑压力、成交量、趋势结构、缠论等技术分析知识结构化。
2. 让后续 V12.2 AI 分析可以引用明确来源，而不是凭空输出。
3. 给 Backtest / Paper Trading / Signal Candidate / Execution Intent 提供技术背景。
4. 建立“证据优先”的研究流程。
5. 建立本地可搜索的技术分析知识库。
6. 支持用户手动添加自己的技术分析笔记。
7. 支持用户手动记录开源资料来源、URL、license、摘要和标签。
8. 支持把 Knowledge Chunk 关联到交易、回测、SignalCandidate、PaperOrder、ExecutionIntent。

核心原则：

```text
Technical context before AI.
Evidence before opinion.
Citation before conclusion.
Research before execution.
No advice without context.
No execution from knowledge.
```

## Data Boundary

V12.1 可以使用三类知识。

### 1. Built-in Technical Analysis Seed Knowledge

项目内置的原创短文本知识条目。

这些内容必须是 AlphaPilot 自己整理的原创总结，不要复制大段书籍、文章、课程、公众号或版权不明确内容。

重点 seed categories：

```text
candlestick_analysis
rsi_analysis
macd_analysis
support_resistance
trend_structure
volume_analysis
volatility_analysis
pivot_points
vwap_analysis
adx_trend_strength
chan_theory
multi_timeframe_analysis
indicator_confluence
pattern_validation
backtesting
risk_management
trade_discipline
execution_boundary
general
```

### 2. User Research Notes

用户手动输入的研究笔记。

例如：

```text
我对某个 RSI 背离案例的理解
我对某个支撑位失效案例的复盘
我对缠论中枢结构的笔记
我对 Paper Trading 里某次止损设计的反思
我对某个 K线组合的观察
```

### 3. Public Reference Metadata

可以记录公开资料的 metadata，例如：

```text
title
author
url
license
sourceType
summary
tags
addedAt
```

V12.1 第一版不要求自动抓取网页全文。

如果实现 URL 字段，只保存 URL 和用户手动摘要，不自动爬取页面。

## Copyright / License Boundary

V12.1 不要复制外部资料的大段原文。

对外部资料只保存：

```text
source title
author / project
url
license
sourceType
trustLevel
summary written by AlphaPilot or user
tags
citationText
short original knowledge chunks written by AlphaPilot or user
```

不要保存：

```text
大段文档全文
完整文章复制
付费课程内容
版权不明确的长文本
外部项目源码
外部项目大段 README
缠中说禅原文
公众号长文
课程讲义
交易社群内部资料
```

所有 KnowledgeChunk 的 `content` 必须是 AlphaPilot 或用户自己整理的原创摘要，不是复制粘贴原文。

缠论特别规则：

```text
缠论部分只保存用户原创笔记框架、术语解释和案例观察。
不要直接复制缠中说禅原文、课程、公众号文章或版权不明确教程。
```

## Future Compatibility

V12.1 为 V12.2 做准备。

V12.2 可以做：

```text
Real AI Analysis Copilot Foundation
```

但 V12.2 必须基于 V12.1 的 Technical Analysis Knowledge Base 做 citation-aware AI analysis。

V12.2 的 AI 应该可以回答：

```text
为什么 RSI 不是单独信号？
当前 Signal Candidate 里的 RSI 应该怎么解释？
支撑位/压力位是怎么来的？
K线形态有没有确认？
这个 Paper Trade 的止损是否和 ATR 匹配？
这个 Backtest 里的 RSI 策略是否可能过拟合？
缠论角度这个结构应该记录哪些 evidence？
```

V12.2 的 AI 不能回答：

```text
现在该买吗？
现在该卖吗？
应该做多吗？
应该做空吗？
下一单怎么下？
```

V12.2 AI 边界必须是：

```text
AI can summarize.
AI can explain.
AI can cite knowledge chunks.
AI can identify technical context.
AI can identify risk.
AI can generate research reports.
AI cannot provide buy/sell advice.
AI cannot create orders.
AI cannot approve Execution Intent.
AI cannot run dry-run.
AI cannot execute trades.
AI never owns money.
```

V12.1 只完成 Technical Analysis Knowledge Base。
