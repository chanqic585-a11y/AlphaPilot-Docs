# AlphaPilot V12.2 Patch - Chinese Mobile UI Polish

## Goal

基于当前已经完成并通过验收的 `trade-discipline-journal` V12.2，做一个小补丁。

本补丁不是重新开发 V12.2，不改 AI 架构，不改数据库主结构，不改交易边界。

本补丁目标是：

```text
让 V12.2 AI 分析页面更适合中文用户手机端实测
把剩余英文 UI 标签中文化
补全用户反馈按钮
优化情景推演中文表达
优化 Research Dashboard AI 区块中文显示
让用户实测时更容易快速判断 AI 报告质量
```

当前 V12.2 已完成：

```text
AI Provider
Mock Provider
OpenAI-compatible Provider
AI Context Builder
Knowledge Retrieval
Habit Analyzer
Prompt Builder
Response Parser
AI Runner
AIAnalysisScreen
AI Analysis database tables
Research Dashboard AI integration
README V12.2 section
```

不要推翻这些实现。

## AIAnalysisScreen Chinese UI Patch

### Header

把页面优化为：

```text
V12.2 AI 决策前分析
AI 技术分析助手
输入币种后，AI 会结合本地行情、技术指标、知识库和交易习惯，生成中文结构化分析报告。
```

### Safety Copy

安全文案更短、更适合手机阅读：

```text
AI 负责分析和整理证据，最终判断和实际操作由用户完成。
本报告是技术研究结果，不是自动交易指令。
API Key 仅用于本次请求，不会写入 SQLite。
```

不要堆叠太多英文安全声明。

### Input Section

Section title：

```text
币种与 AI 配置
```

字段中文化：

```text
Symbol -> 币种
Timeframe -> 周期
Provider -> AI 提供方
Model -> 模型
Base URL -> 接口地址
Runtime API Key -> 本次运行 API Key
```

placeholder：

```text
例如：BTC、ETH、SOL、BTC/USDT
```

按钮：

```text
保存 Provider 配置 -> 保存 AI 配置
开始 AI 分析 -> 开始分析
分析中... -> 分析中...
```

Provider 选项：

```text
Mock -> 本地 Mock
OpenAI-compatible -> OpenAI 兼容接口
```

Timeframe label 为：

```text
周期
```

### Current Report Section

Section title：

```text
AI 分析报告
```

统计卡 label：

```text
Bias -> 技术偏向
Strength -> 偏向强度
Confidence -> AI 置信度
Data Quality -> 数据质量
```

报告字段中文化并优化顺序：

```text

市场概况
K线结构
趋势结构

ATR / 波动率

重点观察
交易习惯观察
引用证据
数据限制
边界说明
```

改为：

```text

```


改为：

```text

```

```text

```

### Limitations Section

当前：

```text

```

改为：

```text

```

### Feedback Section

补全反馈按钮：

```text
有帮助
太模糊
太保守
太激进

手机页面不好读
```

对应 feedbackType：

```ts

'indicator_weak'

'mobile_readability_bad'
'too_slow'
'other'
```

反馈按钮可以分成两行或多行，适合手机点击。

短标签可以使用：

```text

```
