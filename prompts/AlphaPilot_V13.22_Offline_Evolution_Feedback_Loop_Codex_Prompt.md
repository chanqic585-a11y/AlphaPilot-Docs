# AlphaPilot V13.22 - Offline Evolution Feedback Loop

## Goal

Close the auditable research loop from immutable replay, local forward, OKX
Demo, and future Live outcomes back into offline failure diagnosis and bounded
shadow challenger creation.

## Requirements

1. Keep replay, local forward, Demo, and Live outcomes in distinct evidence
   classes.
2. Quarantine probe, synthetic, unknown, integrity-invalid, and lineage-invalid
   outcomes; never use them for performance or evolution.
3. Report expectancy, profit factor, fixed-2R exits, costs, concentration,
   ambiguity, and chronological factor decay.
4. Materialize research-only triggers from formal evidence.
5. Run bounded factor generation and correlation filtering only with real
   registered inputs and materialized factor values.
6. Champion/challenger decisions can request shadow review only.
7. New StrategyCandidates require complete point-in-time, purged walk-forward,
   experiment, FactorRun, fixed `>=2R`, and formal gate evidence.
8. Capture release hashes before and after; no running release may mutate or be
   replaced by the loop.
9. No API key, Trade API, Withdraw API, account/position access, order creation,
   automatic promotion, online model mutation, or fabricated evidence.

## Acceptance

- Probe-only data produces `blocked_no_formal_feedback_evidence`.
- Formal failing outcomes create deterministic research triggers.
- Bounded tests can register only shadow assets.
- Forward/Demo/Live release hashes remain identical across the loop.
- Real reports reflect the registry state without inventing missing evidence.
