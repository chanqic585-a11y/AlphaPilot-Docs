# 3. Core Goal

V13.4.22 的核心目标：

```text
1. 读取 V13.4.21 的 FactorDataPanel 输出或重新生成 factor panel
2. 构建 forward return 标签
3. 构建 TP/SL 先触发标签
4. 对每个手工因子计算 IC / RankIC
5. 计算 top-bottom spread
6. 计算因子分位数组合表现
7. 计算 TP before SL probability
8. 计算 Profit Factor / Expectancy
9. 按 pair / month / regime / universe membership 分析稳定性
10. 输出 factor evaluation report
11. 选择 research-only candidate factors
12. 不生成交易信号
13. 不运行 Freqtrade backtest
```

一句话目标：

```text
先验证因子有没有统计优势，再决定是否用它们生成策略。
```

---

# 12. IC / RankIC Requirements

IC：

```text
corr(factorValue, forwardReturn)
```

RankIC：

```text
corr(rank(factorValue), rank(forwardReturn))
```

必须按 timestamp 横截面计算，再汇总均值/中位数。

不要把所有 rows 混成一个全局相关系数作为唯一结果。

输出：

```text
meanIC
medianIC
positiveICRatio
meanRankIC
medianRankIC
positiveRankICRatio
```

---

# 13. Quantile Analysis

每个因子按横截面分位数分组。

建议：

```text
quantiles = 5
```

输出：

```text
Q1 forward return
Q2 forward return
Q3 forward return
Q4 forward return
Q5 forward return
Q5-Q1 spread
monotonicityScore
```

如果因子 expectedDirection 是 negative：

```text
也要记录方向一致性。
```

不要自动翻转因子方向，除非报告中明确：

```text
factorDirectionAdjusted = true
```

第一版建议不自动翻转，只报告方向。

# 15. Candidate Factor Selection

本版本可以输出 research candidate factors，但不能用于交易。

候选标准建议：

```text
coveragePct >= 80
validSampleCount >= 1000
abs(meanRankIC) >= 0.02
positiveRankICRatio >= 0.55
topBottomSpread > 0
profitFactor > 1.05
```

更严格候选：

```text
abs(meanRankIC) >= 0.03
profitFactor >= 1.15
stableAcrossMonths = true
stableAcrossPairs = true
```

所有候选必须标记：

```text
research_only
not_trade_ready
not_dry_run_ready
```
