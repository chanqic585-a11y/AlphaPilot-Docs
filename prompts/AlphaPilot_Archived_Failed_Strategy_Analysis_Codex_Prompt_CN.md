# AlphaPilot 已归档失败策略批量归因分析 —— Codex 完整提示词（中文代号版）

> 用途：让 Codex 系统分析 AlphaPilot 已经归档或暂停主线推进的失败策略，找出它们究竟为什么失败，区分“信号没有优势”“成本吞噬”“仓位与杠杆放大”“市场状态不匹配”“币种池错误”“方向错误”“退出逻辑错误”“数据或实现问题”等不同原因。
> 本任务不是继续生成新策略，也不是马上调参重跑，而是把过去的失败策略变成可复用的研究资产、负样本规则、基准策略和未来复活候选。

# 七、不要只看账户总收益

很多历史策略出现：

```text
-90%
-99%
接近归零
```

这可能同时包含：

```text
信号质量差
仓位过重
杠杆放大
复利亏损
同时持仓过多
交易成本
频率过高
```

因此必须将失败拆成两层：

```text
A. 信号层失败
B. 账户与风险模型层失败
```

不要仅凭总收益就认定：

```text
策略信号本身完全无效
```

# 八、信号级别归因

如果逐笔交易或候选信号数据可用，请计算或提取：

```text
每笔交易的 R 倍数
未来先到 +1R / +2R 还是 -1R
最大浮盈 MFE
最大浮亏 MAE
入场后立即反向比例
平均有利波动
平均不利波动
止损前是否曾出现足够浮盈
盈利交易平均 R
亏损交易平均 R
信号期望值
```

推荐计算：

```text
单笔 R = 实际盈亏百分比 / 初始止损距离
期望 R = 胜率 × 平均盈利 R - 亏损率 × 平均亏损 R
```

如果没有原始止损距离：

```text
使用报告中的策略止损参数
并标记“估算 R”
```

如果无法计算：

```text
信号级别分析不可用
不要伪造
```

必须回答：

```text
信号本身是否有正期望？
还是信号本身已经为负？
```

# 十八、失败严重程度

建议使用中文等级：

```text
灾难性失败
严重失败
无明显优势
成本敏感
局部可能有效
证据不足
实现问题，不能评价
```

参考标准：

## 灾难性失败

```text
最大回撤接近 100%
总收益接近归零
或大规模长期亏损
```

## 严重失败

```text
PF 明显低于 1
回撤很高
多个维度持续失败
```

## 无明显优势

```text
PF 接近 1
滑点后转负
长期不稳定
```

## 局部可能有效

```text
某些币种 / 方向 / 市场状态存在较稳定优势
但全样本不合格
```

# 二十、负样本规则库

生成新的或合并现有：

```text
负样本规则库
```

至少包括：

```text
1. 不要用宽松条件进行高频做空。
2. 不要把 EMA/RSI/MACD/BOLL 简单组合当成天然优势。
3. 不要在没有成本优势时继续增加交易频率。
4. 不要把小样本正收益当成泛化成功。
5. 不要让做多和做空混合结果掩盖单方向失败。
6. 不要通过删除亏损币重写历史结果。
7. 不要在熊市让 long-only 大量暴露。
8. 不要在修复/牛市让 short-only 大量暴露。
9. 不要在信号本身为负时只修改仓位。
10. 不要在信号为正但账户曲线为负时错误归因策略逻辑。
11. 不要忽略手续费、滑点和价差。
12. 不要把 0 交易策略当成收益失败。
13. 不要无限小调灾难性失败策略。
14. 不要采用马丁、无限加仓或摊平。
```
