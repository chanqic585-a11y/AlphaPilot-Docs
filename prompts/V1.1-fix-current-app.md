# V1.1 fix current app prompt

## Goal

基于当前 `trade-discipline-journal` 仓库完成 V1.1 优化，让现有 App 更稳定、更清晰，并继续保持本地优先和纪律管理定位。

## Scope

本阶段只修正当前 App 的基础体验、风控边界、导航结构和启动流程。

包含范围：

- 合约杠杆上限
- 首次启动资金设置
- 底部导航结构
- “我的”页面入口
- Monitor 前台提醒提示
- 创建交易前检查清单
- 首页纪律评分
- SQLite 本地存储
- TypeScript 检查

## Requirements

1. 将所有合约杠杆上限设为 5x，包括 `TradePlanScreen` 校验、README、文案和类型说明。
2. 首次启动增加 Initial Setup 页面，让用户设置初始本金，默认建议值 15000；设置后写入 `AccountSettings`。
3. 简化底部导航为 4 个 Tab：首页、计划、复盘、我的。
4. 将 Monitor、Statistics、Rules 移入“我的”页面中的卡片入口。
5. 在 Monitor 页面顶部增加醒目提示：当前版本只在 App 前台运行时提醒，锁屏或后台可能停止监听。
6. 在创建交易计划前增加 Pre-Trade Checklist。
7. 全部勾选 Pre-Trade Checklist 后才允许保存。
8. 首页增加“纪律评分”，根据今日是否复盘、是否违规、是否连续亏损、是否移动止损等计算 0-100 分。
9. 运行 TypeScript 检查并修复所有报错。

Pre-Trade Checklist 包含：

- 我不是为了回本开单
- 我不是因为害怕错过
- 我已经设置止损
- 本单风险不超过账户 2%
- 这笔交易符合我的形态系统
- 我已准备好接受止损

## Safety Rules

- 保持本地 SQLite 存储
- 不接交易权限
- 不保存交易权限 API Key
- 不保存提现权限 API Key
- 不自动下单
- 不提供买卖建议

## Acceptance Criteria

- 合约杠杆最高只能设置为 5x
- 首次启动可以设置初始本金，默认建议值为 15000
- 底部导航只显示：首页、计划、复盘、我的
- Monitor、Statistics、Rules 可以从“我的”页面进入
- Monitor 页面显示前台提醒限制
- Pre-Trade Checklist 未全部勾选时不能保存交易计划
- 首页显示纪律评分
- `AccountSettings` 正确写入初始本金
- TypeScript 检查通过
- README 同步更新
