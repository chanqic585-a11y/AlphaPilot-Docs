# AlphaPilot V13.1 - Realtime Market Watchlist and Analysis Data Reliability

## Version Name

```text id="osv4gx"
AlphaPilot V13.1 - Realtime Market Watchlist and Analysis Data Reliability
```

中文版本名：

```text id="xqp340"
AlphaPilot V13.1 - 实时行情观察与 AI 分析数据可信度优化版
```

---

## Background

V13 已经完成主线纠偏：

```text id="lkypb5"
输入币种
获取 OKX 公共实时行情
获取 K线数据
自动计算 RSI、MACD、ATR、成交量、趋势、支撑位和压力位
结合 V12.1 技术知识库
生成中文 AI 技术分析建议
用户自行判断是否交易
开仓后登记交易计划
```

但是用户真机初步测试后发现：

```text id="d6xetp"
1. 获取行情不是持续实时刷新，数据不够准确。
2. 当前是点击“获取行情并分析”时才临时拉取行情。
3. 计划页面和“我已开仓，登记计划”重复。
4. 旧计划页面 UI 仍有英文，且在主流程里很多余。
5. 手动交易计划保存后不能编辑。
6. 登记计划时没有自动带入 AI 报告里的支撑压力、当前价格等数据。
7. 支撑位 / 压力位算法太粗糙，不够可信。
8. K线周期只有 5m / 15m / 1h / 4h / 1d，不够。
9. AI 分析没有清楚显示本次到底用了什么数据。
10. 很多行情数据没有单位，例如价格、成交量、ATR、支撑压力等。
11. 实时行情价格的小数点位数不够，不能固定用 toFixed(2) 或 toFixed(4)。
12. 后续如果要升级自动交易，必须先有稳定的实时行情层。
```

因此 V13.1 不是自动交易版本。

V13.1 的目标是：

```text id="rvw9dw"
先把行情数据做准、做稳、做实时；
再让 AI 分析明确基于最新缓存行情；
同时精简计划页面，并让开仓后交易计划可以编辑；
最后补齐单位显示、价格精度和数据可信度展示。
```

---

## One-Sentence Goal

```text id="c3fs02"
用户手动设置 1～2 个观察币种后，App 自动刷新 OKX 实时行情和 K线数据，计算指标并缓存；用户需要时再点击 AI 分析，AI 必须基于最新缓存行情、指定 K线周期和分析范围生成中文分析建议，并且所有关键数据必须显示单位、精度和数据更新时间。
```

---

## Core Product Principle

V13.1 必须拆分：

```text id="sjqkq8"
行情自动刷新
AI 分析人工触发
交易执行继续禁止
```

也就是：

```text id="bncxc4"
App 自动获取行情
App 自动更新 K线
App 自动计算指标
用户需要时才点击 AI 分析
```

不要自动调用 AI。

不要自动交易。

不要自动下单。

V13.1 不允许默认固定 BTC、ETH、SOL 或任何具体币种。

观察币种必须由用户自己设置。

首次进入页面时：

```text id="wcby1u"

不自动填 ETH
不自动填 SOL
不启动行情刷新

```

placeholder 可以出现示例：

```text id="k6qf5n"
例如：SOL、BTC、ETH、DOGE、BTC/USDT
```

但示例不能作为默认 state 值。

# Part 5 - Units and Decimal Precision

## Critical Requirement

V13.1 必须修复：

```text id="cl1jhj"
数据没有单位
价格小数位不够
所有价格统一固定 toFixed(2) 或 toFixed(4)
```

所有行情显示、AI 分析数据头、技术指标卡片、交易计划自动带入字段、调试面板，都必须显示单位。

---

## Required Units

### Price

价格单位使用 quote currency。

示例：

```text id="j4neps"
SOL/USDT 永续 最新价格：145.238 USDT
BTC/USDT 现货 最新价格：61325.4 USDT
```

字段：

```text id="x1nkjn"
最新价格：USDT
开盘价：USDT
最高价：USDT
最低价：USDT
收盘价：USDT
支撑区域：USDT
压力区域：USDT
ATR：USDT
MACD：USDT 价格差值
```

---

### Percentage

百分比必须带 `%`。

示例：

```text id="c6psyt"
24h 涨跌幅：+2.35%
成交量变化：1.42x
数据质量：86 / 100
```

---

### Volume

成交量必须区分：

```text id="wbrpak"
基础币成交量
计价币成交量
```

示例：

```text id="hdkhjd"
24h 成交量：128,500 SOL
24h 成交额：18,650,000 USDT
```

如果只有一个 volume 字段，必须标注来源：

```text id="ylp0mw"
成交量：128,500 SOL
```

不要只显示：

```text id="lopk9t"
成交量：128500
```

---

### RSI

RSI 没有价格单位，但必须说明范围：

```text id="dg6xrz"
RSI14：62.35 / 100
```

---

### MACD

MACD 是价格差值类指标，单位跟价格一致。

示例：

```text id="bi3g7f"
MACD：0.0185 USDT
Signal：0.0142 USDT
Histogram：0.0043 USDT
```

如果 UI 空间不足，可以显示：

```text id="gfujel"
MACD：0.0185
```

但详情面板和 AI 数据头必须说明：

```text id="u20cqg"
MACD 单位：USDT 价格差值
```

---

### ATR

ATR 必须带价格单位。

示例：

```text id="lwfx61"
ATR14：2.35 USDT
```

---

### K-line

K线数量必须带：

```text id="o18ksu"
根
```

示例：

```text id="trj83r"
实际K线数量：672 根
```

---

### Time

时间必须中文显示。

示例：

```text id="i83ncm"
ticker 更新时间：2026-07-03 14:35
最新K线时间：2026-07-03 14:30
刷新间隔：10 秒
分析范围：7 天
```

## Instrument Metadata

### Metadata Rules

```text id="wjnn2j"
1. 优先从交易所 public instrument metadata 获取 tickSize / lotSize。
2. 如果获取不到 metadata，使用 fallback 精度规则。
3. 不允许因为 metadata 获取失败导致行情无法显示。
4. 调试面板必须显示 tickSize、pricePrecision、lotSize、sizePrecision。
```

---

## Price Precision Calculation

如果拿到 tickSize：

```text id="h74d7z"
tickSize = 0.1      => pricePrecision = 1
tickSize = 0.01     => pricePrecision = 2
tickSize = 0.0001   => pricePrecision = 4
tickSize = 0.000001 => pricePrecision = 6
```

实现：

```ts id="b7gnbn"
function precisionFromTickSize(tickSize: number | null): number
```

限制：

```text id="f8tzpp"
pricePrecision 最小 0
pricePrecision 最大 12
```

---

## Fallback Price Precision

如果没有 tickSize，使用价格大小动态判断：

```text id="japzvl"
价格 >= 10000：显示 2 位小数
价格 >= 1000：显示 2 位小数
价格 >= 100：显示 3 位小数
价格 >= 1：显示 4 位小数
价格 >= 0.01：显示 6 位小数
价格 >= 0.0001：显示 8 位小数
价格 < 0.0001：显示 10 位小数
```

示例：

```text id="mz0grt"
BTC：61325.42 USDT
SOL：145.2380 USDT
DOGE：0.123456 USDT
PEPE：0.0000123456 USDT
```

不要因为小数位不足，把小币种显示成：

```text id="wxnp30"
0.00 USDT
```

---

## Volume Precision

成交量格式：

```text id="gzjpw6"
大数使用千分位
小数最多 4 位
```

示例：

```text id="qjq78p"

18.65M USDT
0.3254 BTC
```

如果使用 K / M / B，必须中文说明或统一显示：

```text id="m2rftd"

```

### Background

V13.1 的方向是正确的：用户自定义观察币种、自动刷新实时行情、缓存 K线和指标、AI 基于最新缓存数据分析、补齐单位和动态价格精度、计划页面精简并可编辑、支撑压力区域更可信。

### Tab Responsibility

`行情` 页面只负责实时行情观察：用户设置 1～2 个观察币种，自动刷新 ticker、OHLCV 和指标缓存，显示实时价格、K线周期、分析范围、K线数量、数据更新时间、关键指标摘要和行情调试信息。行情页可以有 `刷新行情`、`查看调试信息`、`去 AI 分析`、`登记计划` 按钮，但不自动调用 AI，不默认展示完整 AI 报告。

`AI分析` 页面只负责基于最新行情缓存生成 AI 分析报告：选择一个已设置的观察币种，显示最新缓存状态、本次将使用的 K线周期、分析范围、K线数量和数据更新时间，用户点击按钮后才生成 AI 分析。缓存不存在时提示先去行情页添加观察币种并刷新行情；缓存过期时提示先刷新行情后再分析。

`计划` 页面只负责用户已经手动开仓后的交易记录、编辑、归档和复盘，不做旧版开仓前审批，不自动交易，不创建真实订单。

`我的` 页面负责设置、AI Provider、高级功能、旧版功能、数据管理和长期路线说明。旧版交易计划、旧版复盘、旧版统计、旧版行情监听、Research Dashboard、Backtesting、Signal Engine、Paper Trading、Semi-Auto Design、Feature Database、Skill Engine、旧版 AI 分析都放入高级功能或我的页入口，不作为主 Tab。

```text

我的 -> MyScreen
```

要求：不要让 `行情` Tab 和 `AI分析` Tab 渲染同一个组件。如果需要复用，只能复用行情卡片、指标卡片、报告卡片等小组件。旧版 `TradeWorkspaceScreen` 和 `ReviewTradeScreen` 不再作为主 Tab，必须移入 Advanced Features。

### Basic Key Indicator Highlight

V13.1 补丁需要在行情页和 AI分析页基础高亮：
