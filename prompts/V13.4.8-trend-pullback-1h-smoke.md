# 12. Entry Logic V03

最终入场条件必须体现以下结构。

---

## 12.1 4h Trend Filter

第一版实现：

```text
close_4h > ema200_4h
ema20_4h >= ema50_4h
ema20_4h >= ema20_4h.shift(1)
```

含义：

```text
4h 价格在长期趋势上方
4h 中短均线结构不弱
4h EMA20 斜率不为负
```

---

## 12.2 BTC Market Safety Filter

第一版实现：

```text
btc_1h_return_3 > -0.015
btc_4h_close >= btc_4h_ema200
btc_1h_macdhist >= btc_1h_macdhist.shift(1) 或者 btc_1h_macdhist 连续恶化不成立
```

建议实现方式：

```text
btc_1h_return_3 = btc_close / btc_close.shift(3) - 1
btc_market_safe = btc_1h_return_3 > -0.015
```

如果 BTC 4h data 难以稳定 merge，至少实现 BTC 1h 三根急跌过滤，并在 docs 中说明 BTC 4h filter 为 TODO。

不要完全跳过 BTC safety。

---

## 12.3 1h Pullback Location

第一版实现：

```text
close >= ema50
close >= ema20 * 0.985
close <= ema20 * 1.015
```

含义：

```text
价格仍在 EMA50 上方
价格在 EMA20 附近
不是远离 EMA20 的追高
```

---

## 12.4 Reclaim / Confirmation

第一版实现：

```text
close > ema20
macdhist > macdhist.shift(1)
```

含义：

```text
价格重新站上 EMA20
动能开始改善
```

---

## 12.5 Volume Quality

第一版实现：

```text
volumeRatio >= 1.2
```

说明：

```text
1h 周期下不沿用 15m 的 1.5，先使用 1.2，避免样本过少。
```

---

## 12.6 No-Chase Filter

第一版实现：

```text
close <= ema20 * 1.02
rsi <= 65
```

含义：

```text
避免已经大幅拉高后追入。
```

---

## 12.7 Optional Risk Quality

V13.4.8 可以实现简单风险质量过滤：

```text
atr_pct <= 0.08
```

如果实现会导致信号过少或 runtime 不稳定，可以先 TODO。

如果暂不实现，必须在 docs 写：

```text
risk quality filter deferred
```

# 14. Exit Logic V03 ExitProfileA

## 14.1 Stoploss

```text
stoploss = -0.025
```

## 14.2 ROI

```text
minimal_roi = {"0": 0.05}
```

## 14.3 Time Stop

实现：

```text
入场后 8 根 1h K线仍未盈利，则退出
```

可用 `custom_exit`。

如果 `custom_exit` 难以稳定获取 candle count，可用 trade open time：

```text
持仓时间 >= 8 小时 且 current_profit <= 0
```

则返回：

```text
"time_stop_8h_not_profitable"
```

## 14.4 Profitable Momentum Exit

只在盈利时启用：

```text
current_profit > 0
AND macdhist < macdhist.shift(1)
AND macdhist.shift(1) < macdhist.shift(2)
```

返回：

```text
"profitable_macd_weakness_exit"
```

注意：

```text
V0.1/V0.2 的 MACD 弱退出在亏损时表现很差。
V03 只允许盈利状态下的动能退出。
```
