# 8. Entry Logic V0.1

入场条件：

```text
1. BTC 没有急跌
2. 4h 趋势不处于强空头
3. RSI14 在 30～55
4. volumeRatio >= 1.5
5. MACD histogram 当前值高于上一根
6. close >= EMA20 × 0.995
7. 价格处于回调区域，不能追高
```

建议实现细化：

```text
btcCrashFilter:
  BTC 最近 3 根 15m close 累计跌幅 > -1%
  如果 BTC 3 根 15m 累计跌幅 <= -1%，禁止新 entry

4h trend filter:
  当前交易对 4h close 不明显低于 EMA200
  或者 4h EMA20 没有明显向下远离 EMA200
  V13.3 可以采用简化规则：close_4h >= ema200_4h * 0.98

RSI:
  30 <= RSI14 <= 55

volumeRatio:
  volumeRatio = 当前 15m volume / 最近 20 根 15m volume rolling mean
  volumeRatio >= 1.5

MACD:
  macdhist > macdhist.shift(1)

EMA20:
  close >= ema20 * 0.995

pullback / no chase:
  close <= bb_middleband * 1.01
  或者 close 曾经接近 / 跌破 bb_lowerband 后回收
  V13.3 采用简化规则：close <= bb_middleband * 1.01
```

最终 entry signal 只有全部条件满足才置：

```python
dataframe["enter_long"] = 1
```

# 9. Exit Logic V0.1

出场条件：

```text
1. -3% 固定止损
2. +3% 全部止盈
3. MACD histogram 连续 2 根走弱，提前退出
4. 入场后 12 根 15m K线仍未盈利，时间止损
```

V13.3 可实现：

```text
Fixed stoploss:
  stoploss = -0.03

Take profit:
  minimal_roi = {"0": 0.03}

MACD weak exit:
  macdhist < macdhist.shift(1)
  and macdhist.shift(1) < macdhist.shift(2)

Time stop:
  如果 Freqtrade strategy callback 实现复杂，V13.3 可以先 TODO
  但必须在 docs 里记录 V13.4 / V13.5 需要实现
```

如果实现 custom_exit，需要保持逻辑简单，并在 Self Review 中说明。

V0.1 不做分批止盈。

# 12. Backtest Assumptions

必须遵守以下回测假设：

```text
1. 信号在 K线收盘后确认
2. 下一根 K线开盘价模拟入场
3. 必须考虑手续费
4. 必须考虑滑点
5. 同一根 K线同时触发止盈止损，按保守逻辑记录风险
6. 不能使用未来数据
7. 不能使用今天的热门榜回测过去
```

Freqtrade 本身的策略 dataframe 在回测中会一次性传入完整区间，因此策略实现必须避免未来函数。
