# AlphaPilot V13.27.8 - Shared Official Market Data

## Goal

让不同策略合约共享已经验证的 OKX 官方公共 OHLCV，只下载缺失尾段，
避免相同币种和周期从 2020 年开始重复下载。

## Required Flow

1. 先运行本地非官方数据研究烟测，不参与正式晋级。
2. 按 `exchange / market / instrument / timeframe` 查找共享规范快照。
3. 校验 Manifest 身份、官方 endpoint、规范目录边界、文件存在和 SHA256。
4. 从快照最后一根 K 线之后拉取已闭合尾段。
5. 无新尾段时复用原不可变文件；有尾段时合并并写入新内容寻址快照。
6. 继续执行原正式校验、回测和晋级流程。

该规则统一适用于 `5m`、`15m`、`1h`、`4h`、`1d` 和以后支持的周期。
部分分页检查点只用于展示进度，不能作为共享正式证据。

Alpha191 使用 `4h` 信号，但正式数据契约同时需要 `5m` 执行、`15m`
回退和 `4h` 信号数据，因此应复用三套共享数据。

## Queue Rule

以下三条工作流可以同时处于逻辑队列：

1. `5m 极端超卖收回 ATR1.2`
2. `15m EMA20 回收反弹 ATR1.4`
3. `Alpha191 加密因子观察策略`

物理网络下载保持单 worker 串行，避免 OKX 限流争用。后续策略优先复用
前序策略已经完成的共享分区。

## Non-Goals

- 不修改策略参数。
- 不修改 `targetR >= 2R`。
- 不修改正式晋级证据。
- 不接私有账户接口。
- 不保存 API Key。
- 不接 Withdraw。
- 不新增实盘交易权限。
