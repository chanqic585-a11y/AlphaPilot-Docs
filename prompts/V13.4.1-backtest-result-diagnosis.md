# 3. Core Goal

V13.4.1 的核心目标：

```text
1. 读取 V13.4 真实 smoke backtest report
2. 读取可用的 Freqtrade 原始 backtest result
3. 分析亏损来源
4. 按交易对、月份、退出原因、持仓时间、盈利/亏损、连续亏损等维度诊断
5. 检查手续费 / 滑点 / 成本影响
6. 检查策略过滤器是否可能过宽或失效
7. 输出结构化诊断报告
8. 给出 V0.2 候选修改方向
9. 不直接修改策略参数
```

一句话目标：

```text
先诊断为什么亏，再决定怎么改；不要看到亏损就直接调参。
```

---

# 4. Required Mindset

请遵守以下原则：

```text
1. 先诊断，不调参
2. 先看数据，不凭感觉
3. 不擅自修改策略参数
4. 不把一次 smoke 结果当成最终结论
5. 不因为亏损直接进入过拟合调参
6. 所有结论必须来自报告或原始交易数据
7. 缺失数据必须标注 unavailable，不允许编造
8. 实盘与回测必须严格区分
```

# 11. Diagnostic Dimensions

诊断报告必须尽量包含以下维度。

如果原始数据不足，字段保留 null，并在 warnings 中说明。

---

## 11.1 Overall Metrics

必须读取或计算：

```text
total trades
winning trades
losing trades
win rate
total return
max drawdown
profit factor
average profit
average loss
average win
average loss
expectancy
max consecutive losses
average holding time
median holding time
fees
slippage if available
```

如果滑点未实际应用，必须写：

```text
slippageApplied: false
```

不要假装已经应用滑点。

---

## 11.2 Pair Breakdown

按交易对统计：

```text
BTC/USDT:USDT
ETH/USDT:USDT
SOL/USDT:USDT
```

每个 pair 输出：

```text
tradeCount
winRate
netProfitPct
totalProfit
averageWin
averageLoss
profitFactor
maxDrawdown if available
maxConsecutiveLosses
averageHoldingTime
bestTrade
worstTrade
```

回答问题：

```text
哪个币贡献最大亏损？
哪个币表现相对最好？
亏损是否集中在某一个币？
```

---

## 11.3 Monthly / Time Breakdown

按月份或周统计：

```text
tradeCount
netProfit
winRate
profitFactor
drawdown
```

回答问题：

```text
亏损是否集中在某个月？
是否某段行情特别不适合该策略？
是否每个月都亏，还是某段拖累？
```

---

## 11.4 Exit Reason Breakdown

按退出原因统计：

```text
roi / take profit
stoploss
custom_exit
force_exit
timeout / time_stop
macd_weak_exit
other
```

每类输出：

```text
tradeCount
winRate
netProfit
averageProfit
averageLoss
```

回答问题：

```text
主要亏损来自止损吗？
MACD 两根走弱退出是否有效？
时间止损是否有效？
止盈是否过早或过少？
```

---

## 11.5 Holding Time Breakdown

按持仓时间分组：

```text
0-1h
1-3h
3-6h
6-12h
12h+
```

输出：

```text
tradeCount
winRate
netProfit
averageProfit
```

回答问题：

```text
亏损是否来自持仓过久？
短持仓是否更有效？
12 根 15m 时间止损是否太晚？
```

---

## 11.6 Cost Analysis

分析：

```text
feesPaid
fees / grossProfit
fees / absolutePnL
slippageCost if available
netReturnAfterCosts
```

回答问题：

```text
策略是否被交易成本吃掉？
15m 周期是否交易过频？
手续费是否导致本来微利变亏？
```

如果费用字段不可用：

```text
feesAvailable: false
```

并写 warning。

---

## 11.7 Consecutive Loss Analysis

输出：

```text
maxConsecutiveLosses
lossStreaks
worstLossStreakStart
worstLossStreakEnd
symbolsInWorstLossStreak
```

回答问题：

```text
最大连续亏损来自哪些币？
是否需要连续亏损暂停？
```

---

## 11.8 Filter Effectiveness Review

检查或推断这些过滤器是否有实际效果：

```text
BTC crash filter
4h trend filter
RSI 30-55 filter
volumeRatio >= 1.5
MACD histogram improving
close >= EMA20 * 0.995
pullback / no chase filter
```

如果当前 report 无法统计过滤前后的跳过次数，则输出：

```text
filterEffectivenessAvailable: false
reason: "skipped signal instrumentation not available in V13.4"
```

并建议 V13.4.2 或 V13.5 增加：

```text
signal audit instrumentation
pre-filter signal count
post-filter signal count
skip reason count
```

不要伪造过滤器效果。

---

## 11.9 Trade Quality Review

尽量分析：

```text
average favorable excursion
average adverse excursion
max adverse excursion
whether trades nearly hit TP then reversed
whether trades hit SL quickly
whether entry after pump/chase
```

如果 Freqtrade result 没有这些数据，写：

```text
MFE/MAE unavailable in current result
```

# 14. V0.2 Candidate Ideas

V13.4.1 可以提出 V0.2 候选方向，但不能直接改策略。

候选方向示例：

```text
1. 增强 4h 趋势过滤
2. 提高 volumeRatio 阈值
3. 增加不追高过滤
4. 缩短或移除 MACD 弱退出
5. 调整时间止损
6. 分析 -3% 止损是否过宽
7. 分析 +3% 止盈是否不足
8. 增加多因子 score
9. 降低交易频率
10. 先从亏损最严重的 pair 中排除或单独分析
```

但必须按证据分类：

```text
data_supported
hypothesis_only
needs_more_backtest
not_recommended
```
