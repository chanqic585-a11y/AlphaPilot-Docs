# 10. Universe Selection Factors

每个 snapshot 需要计算以下因子。

---

## 10.1 Liquidity Factors

```text
quoteVolume24h
quoteVolume3d
volumeStability3d
missingCandleRate
```

如果 OHLCV 只有 base volume，没有 quote volume，可估算：

```text
quoteVolume = close * volume
```

并在 report 中记录：

```text
quoteVolumeEstimated = true
```

---

## 10.2 Activity Factors

```text
absReturn24h
absReturn3d
volatility24h
volatility3d
volumeExpansion24h
volumeExpansion3d
```

定义建议：

```text
return24h = close_now / close_24h_ago - 1
absReturn24h = abs(return24h)

return3d = close_now / close_3d_ago - 1
absReturn3d = abs(return3d)

volatility3d = std(hourly returns over last 72 candles)

volumeExpansion24h = quoteVolume24h / averageDailyQuoteVolume over previous 7d
```

如果数据不足：

```text
factor = null
reason = insufficient_history
```

---

## 10.3 Risk Filters

第一版风险过滤：

```text
exclude if missingCandleRate > 5%
exclude if quoteVolume24h <= 0
exclude if quoteVolume3d <= 0
exclude if history available < minimum required
exclude stablecoin-like pairs if present
exclude delisted / unavailable pairs
```

如果没有 delist 数据，则记录：

```text
delistFilterAvailable = false
```

---

# 11. Ranking Formula

V13.4.13 先使用简单可解释评分。

建议：

```text
universeScore =
  0.35 * rankScore(quoteVolume24h)
+ 0.25 * rankScore(quoteVolume3d)
+ 0.15 * rankScore(absReturn24h)
+ 0.10 * rankScore(absReturn3d)
+ 0.10 * rankScore(volatility3d)
+ 0.05 * rankScore(volumeExpansion24h)
```

说明：

```text
rankScore 使用横截面排名归一化，0-100。
越高越好。
```

注意：

```text
不要让涨幅因子权重过高，避免变成追涨榜。
成交额和流动性权重必须最高。
```
