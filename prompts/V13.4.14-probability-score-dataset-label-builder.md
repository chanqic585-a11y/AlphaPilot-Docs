# 3. Core Goal

V13.4.14 的核心目标：

```text
1. 读取历史动态币种池 snapshots
2. 读取本地 OHLCV
3. 为每个 snapshot/pair 生成候选样本
4. 对候选样本计算未来窗口标签
5. 标记未来是否先触发 TP 或 SL
6. 计算 MFE / MAE
7. 按 regime、流动性、波动率、RSI、EMA距离等条件分桶
8. 生成 probability score table
9. 输出 dataset 和 summary
10. 不生成交易订单
```

一句话目标：

```text
用历史数据回答：在类似条件下，未来先到止盈的概率是多少？
```

# 8. Label Definition

默认未来窗口：

```text
8 bars
12 bars
24 bars
```

若主周期是 1h，则对应：

```text
8h
12h
24h
```

标签：

```text
hitTpBeforeSl
hitSlBeforeTp
noHit
mfePct
maePct
futureReturnAtWindowEnd
barsToTp
barsToSl
```

默认 TP/SL 参考：

```text
TP = +5%
SL = -2.5%
```

可配置，但本版本只用于统计，不下单。

---

# 9. No Lookahead Rule

必须遵守：

```text
样本特征只能使用当前 K线及之前数据
标签可以看未来，但只能作为训练/统计标签
标签不能回流到样本特征
不能用未来成交量参与当时 universe ranking
不能用未来走势决定当时入池
```

报告必须说明：

```text
features are point-in-time
labels are forward-looking for evaluation only
```

# 11. Probability Score Table

输出：

```text
reports/v13_4_14_probability_score_table.json
```

每个桶组合输出：

```json
{
  "bucketId": "trend_high_liquidity_near_ema20_btc_safe",
  "sampleCount": 0,
  "hitTpBeforeSlProbability": null,
  "hitSlBeforeTpProbability": null,
  "averageMfePct": null,
  "averageMaePct": null,
  "averageReturnPct": null,
  "profitFactor": null,
  "expectancy": null,
  "confidenceLevel": "insufficient_sample"
}
```

通过条件参考：

```text
sampleCount >= 50
hitTpBeforeSlProbability >= 0.45
profitFactor >= 1.2
expectancy > 0
```

样本不足：

```text
confidenceLevel = insufficient_sample
decision = observe_only
```
