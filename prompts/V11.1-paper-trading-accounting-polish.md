# AlphaPilot V11.1 - Paper Trading Accounting Polish

## Goal

基于当前 `trade-discipline-journal` V11 版本，开发 AlphaPilot V11.1：Paper Trading Accounting Polish。

V11.1 的目标不是新增真实交易，不是半自动交易，不是自动交易，不是交易所接入。

V11.1 的目标是优化 V11 Paper Trading Lab 的模拟账户核算，让纸面账户更接近真实模拟环境。

当前 V11 已经可以：

```text
创建模拟账户
创建模拟订单
生成模拟持仓
生成模拟成交
记录模拟权益快照
更新 mark price
关闭模拟持仓
接入 Research Dashboard
```

V11.1 要进一步优化：

```text
cashBalance
availableBalance
usedMargin
reservedMargin
fee accounting
slippage accounting
open/close PnL
equity
drawdown
position-level accounting
account-level accounting
```

V11.1 仍然只做本地模拟账户核算。

V11.1 不接真实交易权限，不保存交易所 API Key，不读取真实账户余额，不读取真实持仓，不下真实订单，不自动交易，不生成买卖建议。

## Accounting Concept

V11 当前 Paper Trading 已可用，但账户核算比较基础。

V11.1 需要让模拟账户更清楚地区分：

```text
initialEquity 初始模拟权益
cashBalance 模拟现金余额
availableBalance 可用模拟余额
usedMargin 已占用模拟保证金
reservedMargin 预留模拟保证金
equity 当前模拟权益
realizedPnl 已实现模拟盈亏
unrealizedPnl 未实现模拟盈亏
totalFeesPaid 累计模拟手续费
totalSlippagePaid 累计模拟滑点成本
maxDrawdownPercent 最大模拟回撤
```

这些字段都必须明确是 simulated / paper，不能暗示它是真实账户数据。

## Database Migration

V11.1 可以使用 safe migration 给现有表增加字段。

禁止重建旧表、删除旧数据、删除旧表。

### PaperAccounts Add Columns

给 `PaperAccounts` 增加：

```sql
ALTER TABLE PaperAccounts ADD COLUMN availableBalance REAL;
ALTER TABLE PaperAccounts ADD COLUMN usedMargin REAL;
ALTER TABLE PaperAccounts ADD COLUMN reservedMargin REAL;
ALTER TABLE PaperAccounts ADD COLUMN totalFeesPaid REAL;
ALTER TABLE PaperAccounts ADD COLUMN totalSlippagePaid REAL;
```

要求：

1. 先用 `PRAGMA table_info(PaperAccounts)` 检查字段是否存在。
2. 字段不存在才 `ALTER TABLE ADD COLUMN`。
3. 不重建表。
4. 不删除旧账户。
5. 旧账户兼容：`availableBalance` 按 `cashBalance` fallback；`usedMargin`、`reservedMargin`、`totalFeesPaid`、`totalSlippagePaid` 按 0 fallback。

### PaperPositions Add Columns

给 `PaperPositions` 增加：

```sql
ALTER TABLE PaperPositions ADD COLUMN entryFeePaid REAL;
ALTER TABLE PaperPositions ADD COLUMN exitFeePaid REAL;
ALTER TABLE PaperPositions ADD COLUMN entrySlippagePaid REAL;
ALTER TABLE PaperPositions ADD COLUMN exitSlippagePaid REAL;
ALTER TABLE PaperPositions ADD COLUMN grossPnl REAL;
ALTER TABLE PaperPositions ADD COLUMN netPnl REAL;
```

兼容要求：

* `entryFeePaid` null 时，用原 `feePaid` 或 0。
* `exitFeePaid` null 时，用 0。
* `entrySlippagePaid` null 时，用 0。
* `exitSlippagePaid` null 时，用 0。
* `grossPnl` null 时，可以根据 entry/mark 计算。
* `netPnl` null 时，可以用 realizedPnl 或 unrealizedPnl。

### PaperFills Add Columns

给 `PaperFills` 增加：

```sql
ALTER TABLE PaperFills ADD COLUMN grossPnl REAL;
ALTER TABLE PaperFills ADD COLUMN netPnl REAL;
ALTER TABLE PaperFills ADD COLUMN marginReleased REAL;
```

旧 fill 缺字段时按 null 或 0 处理，不修改旧 fill 的原始数据。

### PaperEquitySnapshots Add Columns

给 `PaperEquitySnapshots` 增加：

```sql
ALTER TABLE PaperEquitySnapshots ADD COLUMN availableBalance REAL;
ALTER TABLE PaperEquitySnapshots ADD COLUMN usedMargin REAL;
ALTER TABLE PaperEquitySnapshots ADD COLUMN totalFeesPaid REAL;
```

## Paper Execution Logic

更新 `src/paper-trading/paperExecution.ts`。

### Open Position Accounting

开仓成功时：

```text
notionalValue = filledPrice * quantity
marginUsed = notionalValue / leverage
entryFeePaid = notionalValue * feeRate
entrySlippagePaid = abs(filledPrice - basePrice) * quantity
```

账户更新：

```text
cashBalance = cashBalance - marginUsed - entryFeePaid
usedMargin = usedMargin + marginUsed
availableBalance = cashBalance - reservedMargin
totalFeesPaid = totalFeesPaid + entryFeePaid
totalSlippagePaid = totalSlippagePaid + entrySlippagePaid
unrealizedPnl = sum(open position net unrealized pnl)
equity = cashBalance + usedMargin + unrealizedPnl
```

注意：

* fee 从 cashBalance 扣除。
* margin 从 cashBalance 转为 usedMargin。
* equity 不应因为 margin 占用而减少，只应受 fee 和 PnL 影响。
* rejected order 不占用 margin。
* rejected order 不扣 fee。
* rejected order 不生成 position。
* rejected order 可以保存风险检查结果。

### Mark Price Accounting

更新 mark price 时：

```text
grossUnrealizedPnl = long: (markPrice - entryPrice) * quantity
grossUnrealizedPnl = short: (entryPrice - markPrice) * quantity
netUnrealizedPnl = grossUnrealizedPnl - entryFeePaid
```

账户更新：

```text
unrealizedPnl = sum(open positions netUnrealizedPnl)
equity = cashBalance + usedMargin + unrealizedPnl
availableBalance = cashBalance - reservedMargin
drawdown = based on equity
```

### Close Position Accounting

平仓时：

```text
closeNotionalValue = closePrice * quantity
exitFeePaid = closeNotionalValue * feeRate
exitSlippagePaid = abs(closePrice - baseClosePrice) * quantity
grossPnl = long: (closePrice - entryPrice) * quantity
grossPnl = short: (entryPrice - closePrice) * quantity
netPnl = grossPnl - entryFeePaid - exitFeePaid
```

账户更新：

```text
cashBalance = cashBalance + marginUsed + netPnl
usedMargin = usedMargin - marginUsed
realizedPnl = realizedPnl + netPnl
unrealizedPnl = sum(remaining open positions net unrealized pnl)
totalFeesPaid = totalFeesPaid + exitFeePaid
totalSlippagePaid = totalSlippagePaid + exitSlippagePaid
equity = cashBalance + usedMargin + unrealizedPnl
availableBalance = cashBalance - reservedMargin
```

要求：

* 平仓必须释放 margin。
* 平仓必须写入 close PaperFill。
* PaperFill 需要记录 grossPnl、netPnl、marginReleased。
* PaperPosition 需要记录 grossPnl、netPnl、exitFeePaid、exitSlippagePaid。
* 不调用后端、交易所或真实订单接口。

## Paper Risk Logic

更新 `src/paper-trading/paperRisk.ts`。

风险检查中加入：

1. `marginUsed + entryFeePaid <= availableBalance`。
2. 如果 availableBalance 为 null，则 fallback 到 cashBalance。
3. leverage 仍然不能超过 5。
4. riskPercent 仍然建议不超过 2%。
5. stopLoss 方向检查保持 long: stopLossPrice < entryPrice，short: stopLossPrice > entryPrice。
6. 如果资金不足，订单 rejected。
7. 如果风险超过 2%，订单 rejected。
8. 如果缺少 stopLoss，severity = warning，但不一定 rejected。
9. riskCheckJson 必须写明 simulated account only、no real order、no API key、no exchange account。
