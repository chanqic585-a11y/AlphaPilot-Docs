# 11. Liquidity Gate Design

新增：

```text
alphapilot/execution_reality/liquidity_gate.py
```

定义：

```python
LiquidityGateInput
LiquidityGateResult
evaluate_liquidity_gate()
```

输入字段建议：

```text
symbol
timestamp
marketType
positionNotional
lastPrice
quoteVolume24h
quoteVolume1h
bidAskSpreadPct
orderbookDepthTop5
orderbookDepthTop10
maxPositionToVolumePct
maxPositionToDepthPct
```

输出字段：

```text
approved
decision
reasons
warnings
liquidityScore
estimatedSlippagePct
maxRecommendedNotional
```

默认规则建议：

```text
如果 quoteVolume1h 不可用：warning，不直接 approve
如果 positionNotional > quoteVolume1h * 0.001：reject 或 needs_review
如果 bidAskSpreadPct > 0.0008：reject 或 needs_review
如果 orderbookDepthTop5 可用且 positionNotional > top5Depth * 0.10：reject
```

注意：

```text
V13.4.11 只做规则 skeleton，不接真实盘口。
```

如果数据缺失：

```text
approved = false
decision = "insufficient_liquidity_data"
```

不要默认通过。

# 13. Order Impact Model Design

```text
alphapilot/execution_reality/order_impact.py
```

定义：

```python
OrderImpactInput
OrderImpactResult
estimate_order_impact()
```

输入：

```text
positionNotional
lastPrice
bidAskSpreadPct
orderbookDepth
quoteVolume1h
quoteVolume24h
```

输出：

```text
estimatedImpactPct
impactLevel: low / medium / high / extreme / unavailable
maxRecommendedNotional
warnings
```

简单模型：

```text
如果 orderbookDepth 可用：
  impact = positionNotional / topNDepth

如果 orderbookDepth 不可用但 quoteVolume1h 可用：
  impact = positionNotional / quoteVolume1h

如果都不可用：
  impactLevel = unavailable
```

不要伪造盘口深度。


定义：

```python
LiveFeasibilityInput

```

评分维度：

```text
backtestQuality
slippageRobustness
liquidityQuality
tradeFrequency
pairConcentration
drawdownRisk
lossStreakRisk
executionDataAvailability
shadowTradingReadiness
riskGateReadiness
```

总分：

```text
0-100
```

建议等级：

```text
0-39: not_live_feasible
40-59: research_only
60-74: shadow_ready
75-84: dry_run_candidate
85+: controlled_live_candidate
```

当前默认：

```text
任何没有 Shadow Trading 结果的策略，最高只能到 research_only 或 shadow_ready。
```

不得直接给 dry_run_candidate。
