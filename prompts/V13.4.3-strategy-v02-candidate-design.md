# 9. Evidence To Use

候选设计必须基于以下证据。

---

## 9.1 SOL Overexposure / SOL Loss

证据：

```text
SOL 是最大亏损交易对：-94.83085485 USDT
SOL final entries / actual trades = 135 / 99
```

可推导候选方向：

```text
1. pair-level risk watchlist
2. pair exposure cap
3. per-pair cooldown
4. 不直接永久排除 SOL，只先做观察和对比
```

---

## 9.2 Stop Loss Losses

证据：

```text
stop_loss 是最大退出原因亏损：-420.36251129 USDT
```

可推导候选方向：

```text
1. 入场质量不足导致止损频繁
2. 可能需要更强趋势过滤
3. 可能需要更强 no-chase / risk quality
4. 暂不直接修改 -3% stoploss，先设计候选
```

---

## 9.3 MACD Weak Exit Losses

证据：

```text
macd_histogram_two_candle_weakness 显著亏损：-345.28583144 USDT
```

可推导候选方向：

```text
1. MACD 弱退出可能太噪
2. MACD 弱退出可能在亏损区间频繁退出
3. 候选：MACD 弱退出仅在盈利时启用
4. 候选：移除 MACD 弱退出，用固定 ROI / stoploss / time stop
5. 候选：MACD 弱退出需要结合 EMA20 / RSI / 持仓时间
```

---

## 9.4 Holding Time 1-3h Weakness

证据：

```text
最弱持仓桶：1-3h，-120.76496941 USDT
```

可推导候选方向：

```text
1. 当前时间止损可能过晚或无效
2. 15m 策略在 1-3h 区间出现大量失败
3. 候选：更早识别失败反弹
4. 候选：缩短 time stop
5. 候选：若入场后 N 根 K线未向上，则提前退出
```

---

## 9.5 4h Trend Filter Evidence

证据：

```text
top skip reason: weak_4h_trend，11926
但最终通过的 305 个 entry 仍然整体亏损
```

可推导候选方向：

```text
1. 4h 过滤器有作用，但通过标准可能仍偏宽
2. 候选：从 close_4h >= ema200_4h * 0.98 改为 close_4h >= ema200_4h
3. 候选：增加 4h EMA20 slope / trend direction
4. 候选：只允许 4h close >= EMA20 或 EMA20 > EMA200
```

注意：

```text
V13.4.3 只设计候选，不实际改 V0.1。
```

---

## 9.6 No-Chase Filter Not Blocking

证据：

```text
least primary-blocking filter: no_chase_filter，0
```

可推导候选方向：

```text
1. no-chase 可能太弱
2. no-chase 可能被前面过滤器遮蔽
3. no-chase 可能没有作为 primary reason 出现但仍可能参与 failed_filters
4. 候选：加强 no-chase
5. 候选：调整 skip reason 优先级，单独统计 no-chase fail
```

---

## 9.7 Fee Impact

证据：

```text
手续费约 339.82529451 USDT
滑点未应用
```

可推导候选方向：

```text
1. 策略交易频率可能过高
2. 15m 策略必须降低无效信号
3. V0.2 对比回测必须加入滑点模型
4. 候选：提高信号门槛减少交易次数
```
